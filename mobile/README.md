# Köyden Şehre — Mobile App (Flutter MVP)

Üreticilerle alıcıları doğrudan buluşturan komisyonsuz platformun mobil
uygulaması. Komisyon, ödeme, sipariş, kargo ve uygulama içi mesajlaşma yoktur.

## Bootstrap (one-time)

This repo ships with `lib/`, `pubspec.yaml`, and the manifest snippets you'll
need. Platform scaffolding (`android/`, `ios/`, `web/`, etc.) is generated by
Flutter:

```bash
cd /Volumes/KINGSTON/koydenmsehıre/koyden-sehire-mobile-app

# 1. Generate platform folders without overwriting lib/
flutter create --org com.koydensehire --project-name koyden_sehire .

# 2. Install packages
flutter pub get

# 3. Patch the Android manifest with the permissions in
#    docs/android_manifest_snippet.xml (INTERNET / CAMERA / READ_MEDIA_*)
#    and the deep-link intent-filter for koydensehre.com/apply.

# 4. Patch ios/Runner/Info.plist with the keys in
#    docs/ios_info_plist_snippet.xml (NSCameraUsageDescription, etc.)

# 5. Run against your backend
flutter run --dart-define=BASE_URL=http://10.0.2.2:8080/api/v1   # Android emulator
flutter run --dart-define=BASE_URL=http://localhost:8080/api/v1  # iOS simulator
```

## Architecture

- `lib/app/` — root `MaterialApp.router`, theme, GoRouter config, constants
- `lib/core/` — Dio client, interceptors, secure storage, error mapping, utils
- `lib/shared/` — reusable widgets, extensions
- `lib/features/`
  - `splash/` — initial routing
  - `auth/` — phone + password login (farmer / admin)
  - `otp/` — phone verification (used by application flow)
  - `public/` — home, products, product detail, farmer profile (no-auth)
  - `farmer_application/` — invite → OTP → multi-step application → success
  - `farmer_panel/` — dashboard, profile edit, my products, invitations

State management is `flutter_riverpod` without code generation
(manual providers; manual `fromJson`/`toJson` per clarification #10).

## What is intentionally **not** here (MVP scope)

- Admin mobile screens — admin login lands on a single
  `AdminWebOnlyScreen` with logout.
- Push notifications.
- Offline caching beyond the in-memory category cache + the connectivity
  banner.
- `GET /auth/me` — backend may not implement it; we trust the stored token
  and clear session on the first 401 from any authenticated call.
- Application document upload (no backend endpoint per clarification #7).

## Backend

Backend lives in the sibling `koyden-sehire-mobil/` directory and is **not**
modified by this project. See `koyden-sehire-mobil/docs/` for API reference,
auth flow, and upload contract.
